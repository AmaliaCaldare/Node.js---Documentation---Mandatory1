Server-side development

In order to access web pages of any application, a web server is needed to handle all the HTTP requests.

Node.js allows developers to create and implement different server-side tools in JavaScript, without the usage of another programming language.

Express

Express is a web application framework for Node.js development that provides a set of features to develop a web applcation.

Some of the core capabilities of Express are to handle HTTP requests at different URL paths and set common web application settings, such as ports used or serve static folders.

Installing Express

Express is installed with the usage of the famous npm command, as with any other package or by manually adding the dependency in the package.json file, by writing the name and the desired version.

                
                    npm install express

Importing the Express module

After installing Express and in order to use its capabilities, you need to import the main Express module from the installed package in your Node.js code file.

This can be done by using the require() function and calling the returned object to create the Express application, as following:

                 
                    const express = require("express");
                    const app = express();

Create and handle the HTTP requests

Express includes functions for handling the CRUD operations of a RESTful API:

  • GET: get resource from the server
  • POST: create new resource on the server
  • PUT: update a resource from the server
  • DELETE: delete a resource from the server 
                 
                  app.get('/path', (req, res) => {return res;})
                  app.post('/path', (req, res) => { return res;})
                  app.put('/path/:id, (req, res) { return res;})
                  app.delete('/path/:id', (req, res)) { return res;})

All the above functions contain two parameters.

The first parameter represents the URL or the path to which the GET method should connect.

The second parameter is called a call-back function and it contains two arguments: a request and a response object.

req represnts the request sent to the server and res the response that will be returned to the client.

THE PATH

In order to send data using HTTP requets an URL is needed to define and identify the data in the web - the endpoint URL.

The path represents the last component of an URL and it identifies the resource you are requesting for.

Paths allow you to link to different parts of a website.

To filter or organize the content requested in HTTP method, the path can be combined with values called URL Parameters, such as:

  • Path Variables: should be used when you want to identify an item 
                             
                            "users/:id"
  • Query Strings : should be used when you want to sort or filter items 
                             
                           ".users?name=value"

Example of GET request using Express

                 
                  app.get('/', (req, res) => {
                      res.send("This is a GET requests example");
                  })

TIP: '/' is used to define the root of the website.

Starting the server

After learning how to set up a request, another essential functionality provided by Express is to start the server.

To start a server you need to pass the port number to a listen() function, which will tell our app on which port to run.

When developing it is recommended to use port number 3000 and for production port 80.

TIP: Make use of the cross-env package to support crossing operating systems and avoid errors.

Here is an example on how to start a server:

             
              app.listen(3000, (error) => {
                if(error) {
                    console.log("Error running the server");
                }
                else {
                    console.log("Server is running on port", 3000);
              })

The first parameter of the function represents the port number, while the second one is optional, but commonly used to receive feedback on how the application is running.

Serving static files

Express provides a static() function used to server static files to your web application.

The static files can be images, CSS or other JavaScript file, usually stored in a directory.

                 
                  app.use(express.static('public'));

NODEMON

nodemon is a tool used to develop node.js applications by automatically restarting the application when file changes are made.

nodemon is a replacement for the node command used to run a file and it re-runs the application instead of doing it manually.

To install nodemon globally, type:

            
                npm install -g nodemon
